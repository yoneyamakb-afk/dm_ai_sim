# Rule Compatibility

この文書は、現在の `dm_ai_sim` がデュエル・マスターズ風ルールをどこまで再現しているか、どこからが簡略化・独自仕様・未実装かを棚卸しするための設計監査メモです。

現時点の結論: `dm_ai_sim` は強化学習用の簡略環境としては一貫していますが、実カード大会評価用としてはまだ未完成です。大会用デッキ評価へ使うには、複数ブレイク、スピードアタッカー、G・ストライク、進化、ツインパクト、カード個別能力、レギュレーション情報、対象デッキ別の裁定対応が必要です。

## 実装済み

- 山札、手札、マナゾーン、バトルゾーン、墓地、シールド
- ターン進行: メインフェーズ、攻撃フェーズ、ゲーム終了
- ドロー、先攻初回ドロー設定
- 1ターン1回のマナチャージ
- 召喚
- 通常呪文詠唱
- 攻撃: クリーチャー、シールド、プレイヤー
- シールドブレイク
- シールド0枚の相手へのプレイヤー攻撃勝利
- クリーチャーバトル
- ブロッカー
- 任意ブロック、`BLOCK` / `DECLINE_BLOCK`
- `pending_attack` によるブロック選択待ち
- S・トリガー簡易処理
- 通常呪文効果: `DRAW_1`, `DESTROY_TARGET`, `GAIN_SHIELD`, `MANA_BOOST`
- S・トリガー効果: `DRAW_1`, `DESTROY_ATTACKER`, `SUMMON_SELF`, `GAIN_SHIELD`
- 文明支払い
- 多色カードのマナタップイン
- `COLORLESS`
- 合法手マスクと `ACTION_SPACE_SIZE=384`
- Gym/SelfPlay環境
- ルール分析ログ: blocker, trigger, spell, mana

## 実ルールに近い部分

- マナゾーンのカードをタップしてコストを支払う。
- 多色カードはマナゾーンにタップ状態で置く。
- 単色と `COLORLESS` はマナゾーンにアンタップ状態で置く。
- 使用カードの文明を各1つ以上支払う必要がある。
- 多色マナ1枚は複数文明を持つが、1枚で2文明分を同時に満たさない。
- ブロッカーは攻撃に割り込む。
- シールドブレイク時にS・トリガーが発動する。
- クリーチャー同士のバトルはパワー比較で破壊される。

## 簡略化済み

- S・トリガーは自動発動で、使用するかどうかを選ばない。
- S・トリガーは文明・コスト支払い不要。
- S・トリガー対象選択は固定的で、複数候補からの選択がない。
- `DOUBLE_BREAKER` は `breaker_count=2` として2枚ブレイクする。T・ブレイカー以上や追加ブレイクは未対応。
- 呪文効果は基本4種類に加え、《フェアリー・ギフト》の次CREATURE召喚コスト軽減のみ対応。
- クリーチャー能力は対象デッキ向けに段階的対応中で、カード固有裁定はまだ限定的。
- マナ支払いは自動で、どのマナをタップするかをプレイヤーが選ばない。
- 攻撃中の割り込み処理はブロックとS・トリガー周辺に限定。
- 効果解決順は簡略化。
- 同時誘発、待機中効果、優先権、チェックタイミングは未整理。
- 任意効果の選択がない。
- 対象耐性、置換効果、継続効果がない。
- デッキは生成式の標準デッキで、実カードではない。
- レギュレーション、殿堂、プレミアム殿堂は未管理。

## 独自仕様

- `Action` と `action_id` は学習環境向けの固定スロット方式。
- 召喚、攻撃、呪文、ブロックの意思決定を簡略Actionに分解している。
- S・トリガーは確率や任意性ではなく自動処理。
- `MANA_BOOST` は山札上をアンタップ状態でマナへ置く。
- `DESTROY_TARGET` は通常呪文用、`DESTROY_ATTACKER` はS・トリガー用として分離。
- 標準デッキは学習・テスト用の合成デッキで、大会デッキではない。

- タマシード
- ツインパクトのカード固有裁定全般
- 超次元、サイキック
- GR、オレガ・オーラ
- 封印、禁断
- メクレイド
- ハイパーモード
- 常在能力
- 誘発型能力

## 優先度

High:

- 実カードデータ対応
- 対象デッキ別能力対応
- カード個別能力
- LOCK / META_EFFECT
- 複数S・トリガーの防御側任意順選択
- 詳細な革命チェンジ/侵略条件
- 進化の詳細条件
- 殿堂レギュレーション
- デッキ互換性レポート

Medium:

- 革命チェンジ
- 侵略
- ニンジャ・ストライク
- タマシード
- メクレイド
- 超次元 / GR
- コスト軽減
- 任意効果選択
- 対象選択の汎用化
- 誘発型能力のキュー化

Low:

- 特定時代・特定デッキにしか関係しない特殊処理
- 非主要フォーマット固有の挙動
- まれな同時誘発順序の完全再現

## 大会用デッキ評価におけるリスク

- 複数ブレイク未対応のため、ビートダウンや大型フィニッシャーの速度評価が大きく歪む。
- スピードアタッカー未対応のため、火文明系の詰め性能が低く見える。
- G・ストライク未対応のため、現代的な受け札評価が低くなる。
- ツインパクト未対応のため、マナ基盤・呪文比率・クリーチャー比率の評価が実デッキとずれる。
- カード個別能力がないため、コンボ、リソース、メタカード、除去耐性の評価ができない。
- 自動マナ支払いはプレイヤースキルや温存判断を表現しきれない。
- 標準デッキは学習用合成デッキであり、現実的な大会デッキの速度・安定性とは異なる。

## 次の推奨順

1. 実カードデータの最小スキーマを導入する。
2. デッキ互換性レポートを実装する。
3. 対象デッキを1-2個に絞り、そのカード能力だけを優先実装する。
4. LOCK、META_EFFECT、ALTERNATE_WIN_CONDITION、複数S・トリガーの任意順選択をHigh優先で対応する。
5. 実カード対応後に学習済みモデルを再学習する。
