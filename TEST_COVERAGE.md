# Test Coverage

この文書は、現時点のテストがどのルールを守っているか、どこが不足しているかを整理します。

## カバー済み

- ゲーム初期化
- ドロー、マナチャージ
- 召喚
- 攻撃
- シールドブレイク
- プレイヤー攻撃勝利
- クリーチャーバトル
- ブロッカー
- 任意ブロック
- `pending_attack`
- S・トリガー
- 通常呪文
- `CAST_SPELL` action id
- `action_mask`
- GymEnv
- SelfPlayEnv
- PPO/MaskablePPO短時間訓練
- Spell系Agentのモデルなし動作
- 旧モデルshape不一致時のフォールバック
- 文明支払い
- 多色タップイン
- `COLORLESS`
- S・トリガーが文明支払い不要で発動すること
- analyze/evaluate系スクリプトの最小実行

## カバー不足

- 複数ブレイク
- スピードアタッカー
- G・ストライク
- 進化
- ツインパクト
- 革命チェンジ、侵略
- タマシード、超次元、GR
- 任意効果の選択
- 複数対象の選択
- 同時誘発、解決順
- 置換効果
- カード個別裁定
- 実カードデータ読み込み
- レギュレーション、殿堂
- デッキ互換性レポート

## 壊れやすいテスト

- action id範囲を直接見るテストは、`ACTION_SPACE_SIZE` 変更時に壊れやすい。
- observation shapeを前提にする訓練・Agentテストは、観測追加時に旧モデルと不整合を起こす。
- HeuristicAgentの優先順位に依存するログ分析テストは、ヒューリスティック改善でタグ頻度が変わりやすい。
- 標準デッキの生成規則に依存するテストは、デッキ改善時に文明・コスト・枚数が変わる。

## 追加すべきテスト

- 実カードデータの読み込みとスキーマ検証
- 未対応能力タグの検出
- デッキ互換性レポート
- W・ブレイカーとS・トリガー複数枚処理
- スピードアタッカーの召喚ターン攻撃
- G・ストライクの任意/自動防御処理
- ツインパクトの上下カードタイプ処理
- 進化元条件と召喚酔い
- レギュレーション別の禁止・制限判定
